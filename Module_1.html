PHP & Web Dev Basics

What's *Actually* Happening When You Load a Website?

Web Page vs. Website

This one's simple, but it's the foundation for everything.

Web Page

This is a *single document*. Think contact.html or about.html. It's just one file you're looking at.

Website

This is the *entire collection* of pages, all linked together under one domain (like google.com). It's the whole "house," not just one "room."

The Real Difference: Static vs. Dynamic

This is the most important concept. "Static" is the old way, "Dynamic" is what PHP makes possible.

Static (The "Poster")

A static site is just plain HTML, CSS, and JavaScript files sitting on a server. When you ask for the page, the server just finds that *exact file* and sends it to you. That's it.

  • The content is *fixed*. It's the same for every single person.
  • To change the page, a developer has to *manually edit the HTML file*.
  • Good for: Simple portfolios, restaurant info pages, etc.

Dynamic (The "Video Game")

A dynamic site is *built* for you on-the-fly. There is no finished HTML file just sitting there. When you ask for the page, the server *runs PHP code*. That code talks to a database, pulls your username, finds your posts, and *then* builds a custom HTML page just for you.

  • The content is *personalized*. Facebook, Gmail, etc.
  • The page is *generated* by code (like PHP, Python, or Node.js).
  • Good for: Literally everything else (social media, e-commerce, blogs).

Client vs. Server (Who Does the Work?)

"Client-side" and "Server-side" just mean *where* the code runs. This is simple: the "Client" is your browser, and the "Server" is the big computer somewhere else.

1. CLIENT (Your Browser)

This is the "customer." It *asks* for stuff. It only runs **Client-Side** code (JavaScript) to make the page you're *already looking at* interactive (like image sliders, form popups, etc.).

→

HTTP Request

"GET /index.php"

←

HTTP Response

"Here's your HTML!"

2. SERVER (The "Kitchen")

This is the "worker." It *does* the stuff. It runs **Server-Side** code (PHP) *before* the page is ever sent to you. It builds the page, gets data from the database, and sends the *final, plain HTML* back to the client.

How a PHP Script *Actually* Works

The most important thing to get is that **the user *never* sees your PHP code.** Their browser only gets the final HTML result.

1. Request

You type example.com/index.php in your browser. Your browser sends a request to the server.

↓

2. Server Sees ".php"

The server (e.g., Apache) sees the .php extension. It *doesn't* just send the file. It says, "Oh, this is a PHP file! I need to *run* this first."

↓

3. PHP Engine Activates

The server hands the file to the PHP "engine." The engine reads the file from top to bottom and *executes all the PHP code* inside the <?php ... ?> tags. This is where it talks to the database, does calculations, etc.

↓

4. HTML is Generated

The PHP code (like echo "<h1>Hello, Remo</h1>";) outputs plain text. The engine takes all that text and all the plain HTML in the file and builds *one, final, plain HTML document*. All the PHP code disappears.

↓

5. Response

The server sends this *newly-generated HTML file* back to your browser. Your browser has no idea PHP was ever involved. It just sees HTML and displays it.

Core Data Types

Variables (which always start with $) are just containers. These are the main "shapes" of data they can hold.

Operators (The "Tools")

These are the symbols you use to *do stuff* with your variables.

Arithmetic Operators

The basics. +, -, *, /, and % (modulus, for finding the *remainder* of a division).

Comparison Operators

Used for `if` statements. == (is equal), != (not equal), > (greater than), <= (less than or equal).

Logical Operators

For combining checks. && (AND), || (OR), ! (NOT).

Control Structures (Making Decisions)

This is how your code stops running in a straight line and starts making choices.

`if` / `elseif` / `else`

This is your bouncer. It checks one condition at a time. "Are you 18? `if ($age > 18)`. No? `else` you can't come in."

`switch`

This is a cleaner `if` statement for when you're checking *one variable* against *many possible values*. "Check `$color`: is it 'red'? 'blue'? 'green'?..."

Loops (Doing Stuff Over and Over)

Loops are your workhorses. They save you from writing the same code a million times.

`for` Loop

The "Counter." Use this when you *know* exactly how many times you need to do something. "Run this code *exactly 10 times*."

`while` Loop

The "Sentry." Use this when you *don't know* how many times. "Keep running this code *as long as* `x` is true." It checks the condition *before* each loop.

`foreach` Loop

The "Array Specialist." This is the easiest and best way to go through an array. "Take this array of `_names_` and do something *for each* one."
